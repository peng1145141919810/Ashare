# 量化研究中枢 V5

这是 **从 V4 推倒重来** 的版本，目标不是继续修补“高级迭代器”，而是先把下面三件事做对：

1. **冷启动一定能产出候选实验**，不再一秒结束。  
2. **研究方向先诊断、再分配预算**，不再只是在同一堆参数上兜圈子。  
3. **候选代码在独立实验室中生成与执行**，失败不会污染主库。

## 和 V4 最关键的差别

### 1. 不再做跨轮去重
V4 最致命的问题是：候选生成会读历史注册表，再把跨轮重复项全部跳过。结果就是——明明你觉得是第一次跑，程序却可能因为状态残留直接生成 0 个候选，然后优雅退出。

V5 改成：
- **只在同一轮内去重**
- 跨轮允许再次验证相近方向
- 冷启动父代固定内置，不依赖历史注册表

所以在正常配置下，V5 冷启动不会出现“1 秒结束、0 候选”的情况。

### 2. 研究控制器先判断问题，再决定预算
V5 先看注册表，诊断：
- 是否样本外 IC 太弱
- 是否训练/测试落差过大
- 是否回撤太高
- 是否模型家族太窄
- 是否特征空间太窄

然后把候选实验预算分配到：
- feature
- model
- training
- portfolio
- risk
- data
- hybrid

### 3. 自动代码实验室真正落地
每个候选实验都会有自己的 `labs/candidate_xxx/`：
- `feature_pack.py`
- `train_override.py`
- `generated_model.py`
- `workspace_validation.json`

这些代码默认用模板生成；如果启用 LLM，就会优先用 LLM 产出，再进行编译/导入验证。

## 能做到什么

- 自动读训练表
- 自动分训练/验证/测试
- 自动训练不同模型家族
- 自动生成新特征包 / 新训练覆盖 / 新模型代码
- 自动产出 `train_summary.json`
- 自动产出 `pred_test.csv`
- 自动产出 `latest_scores.csv`
- 自动构建最新组合与简易回测
- 自动维护 `experiment_registry.csv`
- 自动维护 `strategy_family_state.csv`
- 持续多轮循环

## 还没有做什么

- 不会直接修改你旧版主库脚本
- 不会直接实盘下单
- 不会自动把候选代码合并回主库

这是故意的：V5 先保证研究自动化是真的、可追踪、可回滚。
